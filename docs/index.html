Introduction

Collage is the only Discord bot you need. It handles music stuff and server moderation because we couldn't be bothered to make two bots. We took premium features, stuffed them into this thing, and made it free. Why? Because we can. Get to tracking your music collection and using features you don't deserve.

ATTENTION IDIOTS: Yes, Collage is in active development. That means things change. Often. Don't complain when a command breaks; we're making it better. You're just a beta tester who pays nothing. Adapt.

How to Use

  1. Go click the button in the sidebar. (Yes, the one that says 'Invite').
  2. Use the command prefix @collage to access features.
  3. Type @collage help if you immediately forget everything you just read.

FAQ: Questions We Should Never Have To Answer

Q: How do I invite the bot? Are you blind?

A: Click the "Invite Collage" button located in the sidebar. If you missed it, look left.

Q: I found an issue. How do I report a bug?

A: Join the official Discord Server and use the designated #bug-reports channel. Provide context; we don't guess.

Q: Why did the command I ran fail/stop working?

A: It's either a temporary Discord API hiccup, you typed it wrong (99% likely), or we're fixing something critical and you decided to run the exact command we were working on. Try again in 30 seconds. If it's still broken, see the bug report instructions above. Stop pinging me and hounds. I don't want to talk to you.

Q: Is this bot free?

A: Yes. The bot's core functionality is completely free. We do, however, offer ⭐ Premium features because running this complex infrastructure costs money. You want thousands of indexed files and high uptime? Pay up or shut up. Your choice.

Q: Why does the documentation read like a direct personal insult?

A: Because reading is a privilege, not a right. We spent time coding this monster. You spend five minutes reading the manual. We get to control the narrative. You get to stop asking obvious questions. Seems like a fair trade to us, moron.

Q: I used a command and it returned an error about a database connection. Should I panic?

A: Yes. Panic. Then remember that our database is larger than your entire music collection and sometimes needs a break. We get trillions of requests. Give it 10 seconds. If the command still doesn't work, it's actually broken, but your complaining won't fix the server rack. Report it once and go do something useful.

Q: I'm a developer. Can I contribute to this project?

A: Are you serious? The complexity of our codebase would melt your entry-level laptop. This isn't some quick python script; this is a finely tuned machine. If you want to contribute, send us a massive server donation. Otherwise, stick to cloning repositories with 'Hello World' examples. (If you're serious, dm @xbox360s)

Q: Where do you get all this data/files? Is it even accurate?

A: We don't ask, and you don't need to know. The files are here. The metadata is cross-referenced by a team of obsessive music nerds. If you think something is wrong, you are wrong. Our data is superior. Use it and be grateful.

A: Because they have the most leaks, and therefore, they are the best use of our server resources. We serve the demand. If your obscure indie artist had 10,000 unreleased files, we'd build a database for them, too. They don't. So stop asking and listen to a real artist.
